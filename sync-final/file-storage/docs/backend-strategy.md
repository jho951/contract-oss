# Backend Strategy

`file-storage`는 저장 추상화와 실제 저장 구현체를 분리한다.

## 흐름

1. 저장 요청을 인터페이스로 받는다.
2. path 규칙을 적용한다.
3. backend 구현체에 저장 또는 조회를 위임한다.
4. Spring 연동 계층이 결과를 소비한다.

## 원칙

- API는 저장 방식과 무관해야 한다.
- core는 경로 규칙과 저장 추상화만 책임진다.
- backend 교체는 공개 인터페이스를 깨지 않아야 한다.

## 제외 범위

- 인증
- 정책 평가
- 감사 저장
- 서비스 비즈니스 로직
