# Redis Strategy

`rate-limiter`는 in-memory 제한과 Redis 기반 분산 제한을 함께 지원한다.

## 흐름

1. 요청 식별자를 만든다.
2. IP, 사용자 ID, API 키 기준의 키를 계산한다.
3. in-memory 또는 Redis 저장소에서 허용 가능 여부를 확인한다.
4. 제한을 초과하면 거부한다.

## Redis 사용 기준

- 분산 환경에서는 동일 키에 대해 같은 제한 결과를 가져야 한다.
- TTL은 제한 윈도우와 함께 움직여야 한다.
- 로컬 제한과 Redis 제한은 같은 정책 정의를 공유해야 한다.

## 제외 범위

- 인증 principal 생성
- 정책 평가 엔진
- 감사 저장
- 파일 저장
