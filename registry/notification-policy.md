# notification 보류 정책

이 문서는 `notification` 레포의 현재 상태와 분류 보류 규칙을 정의한다.

## 현재 상태

- `notification`은 아직 작성 전이다.
- 현재는 기능과 책임이 확정되지 않았다.
- 따라서 지금은 1계층 최종 분류표에 고정하지 않는다.

## 보류 원칙

- 기능이 생기기 전까지는 모듈 구성을 선결정하지 않는다.
- 코드가 생기면 그 코드가 platform-security, platform-governance, platform-storage 중 어디에 속하는지 다시 판단한다.
- 알림이 단순 이벤트 전파인지, 감사/정책/보안과 결합되는지 확인한 뒤 분류한다.

## 분류 기준

### platform-governance로 갈 수 있는 경우

- 감사 알림
- 정책 변경 알림
- 운영 추적 알림

### platform-security로 갈 수 있는 경우

- 보안 경보
- 인증/인가 이벤트 알림

### platform-storage로 갈 수 있는 경우

- 저장 완료 알림
- 업로드/다운로드 상태 알림

### 별도 OSS로 유지할 수 있는 경우

- 업무 독립적인 일반 알림 기능
- 실제 서비스가 직접 소비하는 별도 notifier

## 결론

- `notification`은 현재 분류 보류다.
- 실제 코드가 생기기 전까지는 레포 역할을 고정하지 않는다.
