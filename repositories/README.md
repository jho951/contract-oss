# Repository Catalog

## 문서

- [layer1/README.md](layer1/README.md): 1계층 OSS 레포 카탈로그
- [layer2/README.md](layer2/README.md): 2계층 platform 레포 카탈로그

## 역할

- 레포별 GitHub URL을 기록한다.
- 레포별 최신 기준 버전을 기록한다.
- 레포별 책임과 공개 모듈 범위를 기록한다.
- 레포별 현재 문서, 운영 파일, 정리 후보를 기록한다.

## 경계

- 계층 기준과 운영 규칙은 `registry`에서 관리한다.
- 문서, build, publish 구조 기준은 `registry/layer1` 또는 `registry/layer2`에서 관리한다.
- 각 구현 레포의 `docs/*` 원문은 구현 레포에 둔다.
- `repositories`는 규칙이 아니라 현재 레포 카탈로그와 점검 상태를 관리한다.

## 레포별 상태 문서

- 1계층: `layer1/<repo>/README.md`
- 2계층: `layer2/<repo>/README.md`
